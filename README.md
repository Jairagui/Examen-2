# Examen-2
Ejercicio 1
En este ejercicio partí del código original del sistema de notificaciones, donde todo estaba metido en la misma clase y en el mismo método con varios if y elif para cada tipo de aviso, y al final esa clase hacía de todo mostraba el pedido, decidía qué notificación mandar, armaba el mensaje y guardaba el historial.. Dejé OrderNotificationSystem para encargarse de procesar el pedido y de pedir las notificaciones, y saqué la parte de enviar el mensaje a clases separadas EmailNotifier, SmsNotifier y PushNotifier, cada una con su método send y su propio mensaje. Para no estar creando los objetos a mano según el tipo, hice una clase NotificationFactory que recibe el tipo de notificación y devuelve el objeto que toca. Con eso el código queda un poco más ordenado cada clase tiene una responsabilidad más clara, si algún día quiero agregar otro tipo de notificación solo tendría que crear otra clase parecida y ajustarla en la fábrica, y la clase principal ya no depende de if gigantes sino de algo más genérico llamado Notifier. Al final también estoy usando la idea del patrón estrategia, porque tengo varias formas de notificar que se usan igual, y una fábrica sencilla para crear el notificador correcto.





Ejercicio2
En este ejercicio partí del código original del sistema de reportes donde todo se hacía dentro de generate_report con varios if para cada tipo de reporte y el mismo método armaba todo el texto. Lo que hice fue separar la parte que genera el contenido en clases nuevas. Creé una clase base llamada GeneradorReporte y tres clases hijas GeneradorReporteVentas GeneradorReporteInventario y GeneradorReporteFinanciero. En GeneradorReporte usé el patrón Template Method. Ahí dejo fija la estructura general del reporte con las líneas de separación el título y la fecha y dejo que las subclases definan solo el nombre del reporte y el contenido específico. En ReportSystem agregué un método interno que se llama crear_generador_reporte que según el tipo de reporte regresa la clase adecuada y luego llamo al método generar del generador. La parte de formatos pdf excel y html . Con estos cambios cada clase tiene una responsabilidad más clara y es más fácil agregar otro tipo de reporte sin tocar tanto el método principal.
